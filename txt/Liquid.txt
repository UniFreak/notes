Liquid is an open-source, Ruby-based template language created by Shopify

Basic concept
    Tag: programming logic that tells templates what to do, wrapped in {% %}
    Objects/Liquid Variables: contain attributes that are used to display dynamic content on the page
    Filter: modify the output, placed within an output tag {{ }}, separated with a pipe character |
    Handle: every object in Liquid has a handle to access the attributes of it

Types: Liquid objects can return one of six types
    String
    Number
    Boolean: true | false
    Nil
    Array
    EmptyDrop: non-existent object, always has a .empty? attribute equal to ture

Operators
    == != > < >= <= or and contains

Tag
    control flow
        if/elsif/else/endif
            {% if customer.name == 'kevin' %}
                Hey Kevin!
            {% elsif customer.name == 'anonymous' %}
                Hey Anonymous!
            {% else %}
                Hi Stranger!
            {% endif %}
        case/when/endcase
            {% case handle %}
                {% when 'cake' %}
                    This is a cake
                {% when 'cookie' %}
                    This is a cookie
                {% else %}
                    This is not a cake nor a cookie
            {% endcase %}
        unless/enduless
            {% unless product.title == 'Awesome Shoes' %}
                These shoes are not awesome.
            {% endunless %}
    iteration
        for/break/continue(output max 50 results, use paginate tag if more than 50 resultes)
            forloop object is available whin for tag

            {% for i in (1..5) reversed limit:2 offset:2  %}
                {% if i == 5 %}
                    {% break %}
                {% elsif i == 3 %}
                    {% continue %}
                {% else %}
                    {{ i }}
                {% endif %}
            {% endfor %}
        cycle
            <ul>
            {% for product in collections.collection-1.products %}
                <li{% cycle 'group1': ' style="clear:both;"', '', '', ' class="last"' %}>
                    <a href="{{ product.url | within: collection }}">
                    <img src="{{ product.featured_image.src | product_img_url: "medium" }}" alt="{{ product.featured_image.alt }}" />
                    </a>
                </li>
            {% endfor %}
            </ul>
            <ul>
            {% for product in collections.collection-2.products %}
                <li{% cycle 'group2': ' style="clear:both;"', '', '', ' class="last"' %}>
                    <a href="{{ product.url | within: collection }}">
                    <img src="{{ product.featured_image.src | product_img_url: "medium" }}" alt="{{ product.featured_image.alt }}" />
                    </a>
                </li>
            {% endfor %}
            </ul>
        tablerow
            tablerow object availabe whthin tablerow tag

            <table>
            {% tablerow i in (1..10) cols:2 limit:3 offset:3%}
                {{ i }}
            {% endtablerow %}
            </table>
    theme
        comment
            {% comment %}any thing inside will not be ouput or excuted by Liquid{% endcomment %}
        include
            inserts a snippet from the snippets folder of a theme
            the code inside the included snippet will have access to the variables within its parent template
            `with` parameter assigns a value to a variable inside a snippet that shares the same name as the snippet

            {% include 'color' with 'red' %}
        form
            generate form
            form object available within form tag
            parameters
                'activate_customer_password'
                'new_comment'
                'contact'
                'create_customer'
                'customer_address'
                'customer_login'
                'recover_customer_password'

            {% form %}
            {% endform %}
        layout
            loads an alternate template file from the layout folder of a theme

            {% layout 'alternate' %}
            {% layout none %}
        paginate
            paginate and part object available within paginate tag

            {% paginate collection.products by 5 %}
                {% for product in collection.products %}
                    <!--show product details here -->
                {% endfor %}
            {% endpaginate %}
        raw
            output of Liquid code on a page without being parsed

            {% raw %}{{ 5 | plus: 6 }}{% endraw %} is equal to 11
    variable
        assign
            {% assing my_var = 'string' %}
        capture
            captures the string inside of the opening and closing tags

            {% capture my_var %}I am being captured.{% endcapture %}
        increment
            creates a new number variable, and increases its value by one every time it is called
            increment created variable are independent from variables created through assign or capture

            {% increment variable %}
        decrement
            {% decrement variable %}
